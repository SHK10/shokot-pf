Mr.Buffett-kun

Using GAS, Line messaging API, Scraping

Service Overview

I've created a line official account as an application that will help me manage my stock holdings. The backend API created in Express (Node.js) parses the Line messages and accesses the spreadsheet data via the GAS API when stock information is needed. The spreadsheet manages the current situation based on the stock information collected by scraping from many websites and the inputs in it. Additionally, every day at the time when the stock market closes, a job is executed and today's stock results are notified as a daily report. In the future, I would like to implement functions such as linking not only to spreadsheet information but also to social networking services such as Twitter, analyzing trends through machine learning, and notifying users.

Back-end / DB : Node.js(Express), Google App Script, Google Spreadsheet
Environments : Heroku, GAS web app
Others : Line messaging API
